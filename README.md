## Глава 1: Математика SDF и бесконечный город

### Что такое SDF?

SDF — это **Signed Distance Function**, или функция знаковой расстояния. Это математическая функция, которая возвращает расстояние от любой точки в пространстве до ближайшей поверхности объекта. Причём:

- Если точка находится **внутри** объекта — значение отрицательно.
- Если точка находится **снаружи** объекта — значение положительно.
- Если точка **на поверхности** — значение равно нулю.

SDF позволяет нам **описывать формы** в пространстве, а затем использовать эти описания для рендеринга, коллизий, генерации текстур и даже физического моделирования.

В нашем случае, мы будем использовать SDF для генерации и отображения зданий, дорог, парков, и прочих элементов города.

Пример простой SDF для круга:

```cpp
float sdCircle(vec2 p, float r) {
    return length(p) - r;
}
```

Здесь `p` — координаты точки в пространстве, а `r` — радиус круга. Функция возвращает расстояние от точки до окружности. Это позволяет легко проверять, находится ли точка внутри или вне фигуры.

### Генерация городских структур с помощью SDF

Для создания городской среды мы используем комбинирование SDF-функций. Например, чтобы создать **здание**, можно объединить несколько примитивов:

- Основание здания — прямоугольник.
- Окна — круги.
- Балконы — полосы.
- Крыша — треугольник или призма.

Каждый такой элемент описывается своей SDF, а затем объединяется в одну общую функцию, позволяющую определить, находится ли точка внутри здания, на его стене или вне его.

Пример объединения двух SDF:

```cpp
float opUnion(float d1, float d2) {
    return min(d1, d2);
}

float opSubtract(float d1, float d2) {
    return max(d1, -d2);
}

float opIntersection(float d1, float d2) {
    return max(d1, d2);
}
```

Эти операции позволяют строить сложные формы из простых. С помощью таких техник можно создавать **бесконечные города**, которые могут быть генерированы **на лету**, без необходимости хранить все данные в памяти.

### Бесконечный город

Один из ключевых принципов нашей игры — **бесконечный город**. Это значит, что игрок может ездить сколько угодно далеко, и город будет продолжаться. Но как же он генерируется?

Мы используем **процедурную генерацию** на основе **координат игрока**. Когда игрок перемещается, мы вычисляем, какие блоки города нужно сгенерировать. Мы используем **периодическую функцию** и **пространственную декомпозицию**, чтобы обеспечить **плавную генерацию** и **отсутствие артефактов** на границах.

Например, мы можем использовать **шум Перлина** или **шум Брауна** для генерации высот, плотности застройки, направления улиц и т. д. Эти данные затем преобразуются в **SDF-описания** городских элементов.

Таким образом, **город не существует заранее**. Он **рождается вокруг игрока**, по мере его движения. Это делает игру **бесконечной**, но при этом **реалистичной** и **динамичной**.

### Пример: Генерация улиц

Улицы — один из самых важных элементов города. Они задают структуру, направление, и обеспечивают движение игрока и NPC.

Мы можем использовать **SDF-линию** для описания улиц:

```cpp
float sdLine(vec2 p, vec2 a, vec2 b) {
    vec2 pa = p - a;
    vec2 ba = b - a;
    float h = clamp(dot(pa, ba) / dot(ba, ba), 0.0, 1.0);
    return length(pa - ba * h);
}
```

С помощью этой функции мы можем определить, находится ли точка на улице, и насколько близко к ней. Затем мы можем использовать её для генерации **дорожной сети**, **пешеходных зон**, **перекрёстков** и даже **объездных дорог**.

### Заключение

Математика SDF становится основой для создания **реалистичных и бесконечных** городов. Она позволяет нам **не только генерировать**, но и **манипулировать** формами, **рендерить** их, а также использовать их для **физики**, **коллизий** и **интерактивности**.
